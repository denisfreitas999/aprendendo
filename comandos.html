Comandos do git:

  1. pwd : mostra o diretório aberto
  2. cd" espaço /diretório" : entra em um determinado diretório
  3. ls : mostra os arquivos do diretório
  4. mkdir" espaço nome_diretório" : cria um diretório
  5. touch" espaço nome_arquivo": cria um arquivo
  6. git init : cria um repositório local no diretório escolhido
  7. ls -a : Exibe o diretório oculto .git
  8. git status : Mostra se houveram modificações nos arquivos do repositório
  9. git add nome_do_aquivo.tipo : Adiciona um arquivo ao repositório
  10. git add * : Adiciona todos os arquivos ao repositório
  11. git add . : Adiciona todos os arquivos ao repositório
  12. git commit -m "Comentários" . : Serve para identificar o conteiner e armazenar os arquivos no repositório
  13. git log : informa os commits, autor e data da criação
  14. git log --oneline : informa os commits (descrição) em linha com seu respectivo código de identificação
  15. git log --graph : mostra os grafos de commits
  16. git log --oneline --graph : mostra os grafos de commits
  17. git commit -am "Comentários" . : Adiciona o arquivo ao conteiner e cria o commit
  18. git branch : Informa o ramo em que o projeto se encontra
  19. git checkout "código do log --oneline" : Comando de backup, serve para acessar versões anteriores do projeto e alternar entre os ramos
  20. git diff : mostra todas as modificações realizadas nos arquivos, passo a passo, antes de utilizar o commit.
  21. git restore 'nome_arquivo.tipo' : serve para restaurar todas as modificações em um arquivo antes de ser aplicado um commit.
  22. git restore --staged 'nome_arquivo.tipo' : remove o arquivo do conteiner para possíveis modificações e/ou restauração do mesmo
  23. git reset --hard ' 'hash_versão_anterior' : remove os commits retornando-o para versão apontada
  24. git log --oneline --graph --all : Mostra todo o grafo com suas ramificações e merges realizados
  25. git merge nome_ramificacao : Usado dentro do ramo principal(master) para fundir determinado ramo ao projeto principal
  26. git remote add origin https://github.com/usuário_do_github/repositório_do_github : Serve para adicionar o repositório de projetos do git local para o remoto no github
  27. git remote : informa o repositório remoto adicionado
  28. git remote -v : traz informações sobre o repositório remoto adicionado
  29. git push -u origin master : Faz o upload dos arquivos do repositório local para o repositório remoto
  30. git clone "link_do_repositorio_gerado_no_site_do_GitHub" : Cria um clone do repositorio do GitHub na diretório desejado
